Django Components Feature Demo

Define components 

{# components/button.html #}
{% define_component %}
<button>
  hello {{ component_vars.kwargs.name }}
</button>
(Soon) Template-Only Components
All in one file. For components without complex logic, define the template directly in the HTML file using {% define_component %}. The template starts right after this tag. 
# components/button.py
from django_components import Component, register

@register("button")
class MyButton(Component):
    template_file = "button.html"
    js_file = "button.js"
    css_file = "button.css"

# components/button.html
<button>
  hello {{ component_vars.kwargs.name }}
</button>

# components/button.js
console.log('Button loaded!');

# components/button.css
button {
  background-color: #805ad5;
  color: white;
}
Python + HTML/JS/CSS
Separate logic, presentation, and behavior. A single Python class can link to separate HTML, CSS, and JavaScript files, keeping your component's assets organized. Ideal for complex, interactive components. 
# components/button.py
from django_components import Component, register

@register("button")
class MyButton(Component):
    template = """
        <button>
          hello {{ component_vars.kwargs.name }}
        </button>
    """
    js = """
        console.log('Button loaded!');
    """
    css = """
        button {
          background-color: #805ad5;
          color: white;
        }
    """
Inline HTML/JS/CSS
All-in-one Python file. For ultimate convenience with small components, you can define the template, script, and style directly as inline strings. This keeps everything neatly organized in a single Python file.

Template Syntax 

# Standard
{% component "button" %}
  Click me!
{% endcomponent %}

# Custom tag alias
{% button %}
  Click me!
{% endbutton %}

# HTML-like syntax
<c-button>
  Click me!
</c-button>
Your Syntax, Your Way
Use what feels right. Whether the standard {% component %} tag, a custom alias, or (soon) an HTML-like syntax, the choice is yours. 
# Standard
{% component "button" / %}

# Custom tag alias
{% button / %}

# HTML-like syntax
<c-button />
Self-Closing Tags
Less is more. For components without children, use the concise self-closing syntax. 
{% component "table"
    title="Friend list"
    headers=table_headers
    attrs:class="py-4 ma-2 border-2"
/ %}
Passing Data with Keyword Arguments
Simple and explicit. Pass data to your components using familiar keyword arguments. 
# In components/table.html:
<h1>
    {{ component_vars.kwargs.title }}
</h1>
Access Data Directly
No Python boilerplate. For simple components, you can access passed keyword arguments directly in the template via component_vars.kwargs. 
{% component "table"
    title="Friend list"
    headers=table_headers
    attrs:class="py-4 ma-2 border-2"
/ %}
Passing Attributes with attrs:
Clean & intuitive. Pass HTML attributes directly using the attrs:key="value" syntax. Great for adding dynamic classes, IDs, or any other attribute to your component's HTML. 
# In components/table.html:
<table {% html_attrs component_vars.kwargs.attrs %}>
    ...
</table>
Render Attributes with {% html_attrs %}
Forward attributes with ease. The {% html_attrs %} tag renders the dictionary of attributes passed via attrs: into valid HTML attributes. 
{% component "table"
    title="Friend list for {{ user.name }}"
    headers=table_headers
    attrs:class="py-4 ma-2 border-2"
/ %}
Template tags in strings
Mix and match! Use {{ user.name }} (or any template tag) inside a string literal.
Format your strings dynamically right where you need them. 
{% component "table"
    headers=["Name", "Age", "Email"],
    data=[
        {
            "name": "John"|upper,
            "age": 30|add:1,
            "email": "john@example.com",
            "hobbies": ["reading"],
        },
        {
            "name": "Jane"|upper,
            "age": 25|add:1,
            "email": "jane@example.com",
            "hobbies": ["reading", "coding"],
        },
    ],
    attrs:class="py-4"
/ %}
List & Dict Literals
Clear and declarative. Define simple data structures like lists and dictionaries directly in your templates. Great for readable, self-contained components. 
{% component "actions"
    show_delete=(user.is_admin and post.num_comments < 1)
    show_archive=(post.views >= 1000)
    attrs:class="py-4 ma-2 border-2"
/ %}
(Soon) Python Expressions in Templates
You don't need filters for simple logic. Use Python expressions for simple adjustments right where you need them. Perfect for that last bit of flexibility before rendering. 
{% component "table"
    ...default_attrs
    title="Friend list"
    headers=["Name", "Age", "Email"]
    attrs:class="py-4 ma-2 border-2"
/ %}
Spread operator ...
You can "spread" a dictionary of attributes or kwargs into the component tag.
...default_attrs will expand all key-value pairs from default_attrs as arguments. 
{% component "card" %}
  {% fill "header" %}
    Card Title
  {% endfill %}

  {% fill "body" %}
    This is the card content.
  {% endfill %}
{% endcomponent %}
Composable UIs with Slots
Build complex UIs from smaller pieces. Use {% fill %} to populate named slots inside your components. It's like putting LEGOs together. 
# In your template:
{% component "table" %}
  {% fill "row" data="data" %}
    <td>{{ data.row.name }}</td>
    <td>{{ data.row.age }}</td>
  {% endfill %}
{% endcomponent %}

# components/table.html:
<table>
  <tr>
    {% slot "row" row=user %}
      <td>Default: {{ user.name }}</td>
    {% endslot %}
  </tr>
</table>
Slot data
Powerful slot system. A component can pass data to its slots. Access it in the {% fill %} block to create powerful, flexible UIs. 
# In components/card.html:
{% if component_vars.slots.header %}
  <header>
    {% slot "header" / %}
  </header>
{% endif %}
Access Slots Directly
Smart templates. Conditionally render parts of the HTML when a slot is present. component_vars.slots is a dictionary of all the slots that were filled.
